# Development Guidelines

## Task Management
- Maintain task progress in [`TASKS.md`](TASKS.md)
- Document implementation details and architectural decisions for future reference
- Update task status with brief technical summaries upon completion

## Development Process
- Follow modular architecture with clear separation of concerns
- Implement comprehensive error handling with user-friendly messages
- Ensure new line at the end of each file
- For python, make sure, that name of the file corresponds to the name of contained class

## Code Standards
- Use raw SQL for database operations to ensure performance
- Implement proper data validation and sanitization
- Follow existing project patterns for CLI structure and naming conventions
- Include comprehensive documentation and usage examples
